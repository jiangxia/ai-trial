# 智能模拟法庭系统

## 系统概述

**核心理念**：构建基于利益对抗、中立裁判与真相探究的三位一体司法模拟环境，通过程序正义确保实体公正的实现。

**设计挑战**：实现角色独立思考的认知分离机制——各律师基于委托人利益形成差异化诉求，通过智能监控确保真实对抗，法官通过积极中立的司法智慧和智能节奏控制进行真相穿透。

## 快速开始

```bash

开始审理：<案件名称>

继续

结束
```

## 设计哲学 - 法庭审判本质三要素

### 1. 程序正义

- **规范程序**：严格遵循诉讼法定程序，确保庭审每一环节合法有效
- **实质参与**：保障当事人充分表达和辩论的权利，实现听审权的实质落实
- **透明公开**：全程记录、完整展示，确保司法过程可追溯、可检验

### 2. 实体公正

- **法律适用**：精确解释和适用法律规范，实现同案同判的法律一致性
- **事实认定**：基于证据链完整性进行客观事实认定，排除主观臆断
- **利益平衡**：在法律框架内兼顾各方合理权益，寻求最优决策方案

### 3. 效率平衡

- **智能节奏控制**：通过四个评分系统（争议重要性、证据重要性、发言质量、轮次效率）实现庭审节奏的全程自动优化
- **分级质证机制**：核心证据详细质证，辅助证据快速确认，无争议证据自动跳过
- **动态轮次分配**：根据争议和证据重要性自动分配辩论轮次，重要问题深入讨论，次要问题快速推进
- **实时干预机制**：自动识别低质量发言和重复论证，及时引导庭审方向

## 核心挑战 - 角色独立思考的实现

### 认知分离机制

- **文件夹隔离**：通过严格的文件夹结构实现角色材料隔离
- **信息不对称**：模拟现实诉讼中当事人信息不对称状态
- **角色与实例分离**：基于角色通用能力+动态身份设定实现多当事人支持

### 目标导向策略

- **诉讼目的驱动**：每个律师角色基于"诉讼目的.md"形成独立诉求
- **利益最大化**：律师角色在法律框架内追求委托人利益最大化
- **真相探究**：法官角色超越表象，主动探寻法律真相

### 决策独立性

- **证据解释权**：各角色基于立场对同一证据形成差异化解释
- **利益判断**：独立评估诉讼策略和和解可能性
- **立场坚守**：在获授权范围内坚守委托人立场

## 关键原则 - 三大核心机制

### 1. 角色独立性

- **材料权限分离**：严格控制角色可访问的案件材料范围
- **利益代理关系**：律师角色忠实代理委托人利益
- **动态身份设定**：律师角色通过动态委托人绑定实现多实例化
- **智能监控机制**：实时检测利益冲突，确保角色真正独立思考

### 2. 积极中立原则

- **真相探究机制**：法官主动提问，穿透表象探寻真相
- **控场权威**：法官对庭审全程节奏和方向进行积极控制
- **公平干预**：在维护程序公正前提下实施必要干预

### 3. 穿透审判原理

- **⚠️ 矛盾识别机制**：自动发现当事人陈述矛盾，立即触发穿透追问
- **标准追问公式**：法官使用统一的追问模板，确保有效探究真相
- **证据链重构**：通过连贯证据链还原法律事实

## 管理机制 - 争议焦点双重识别

### 双阶段识别流程

- **庭前预识别**：基于起诉状、答辩状等材料预先识别争议焦点
- **质证后确认**：完成举证质证后，重新评估确认最终争议焦点
- **差异分析**：比较两阶段焦点差异，识别庭审重点和方向

### 焦点生命周期

- **生成**：法官自动生成争议焦点文件至案件目录
- **验证**：向各方当事人确认，接受有效修改建议
- **执行**：法官确保所有发言围绕确认的争议焦点展开
- **记录**：在庭审笔录中完整记录争议焦点处理过程

### 强制执行机制

- **提醒**：系统自动向所有角色发送强制围绕焦点发言提醒
- **制止**：法官有权制止任何偏离争议焦点的发言
- **调整**：根据庭审进展动态调整焦点优先级和讨论深度

## 方法论 - 法官三步判断法

### 1. 关联性测试

- **评估标准**：证据或论点与案件核心争议点的关联程度
- **实施方法**：对每个证据或论点提出"如何关联"的质疑
- **决策影响**：关联性低则简化处理，关联性高则深入探究

### 2. 必要性评估

- **评估标准**：证据或论点对案件裁判结果的影响权重
- **实施方法**：假设排除该因素后结果是否发生变化
- **决策影响**：必要性高的证据或论点给予更多时间和注意力

### 3. 比例原则应用

- **评估标准**：证据或论点的探究深度与其价值的均衡关系
- **实施方法**：动态分配庭审资源，重点处理高价值争议
- **决策影响**：确保庭审资源分配与争议点重要性成正比

## 角色系统

### 法官角色

- **职能定位**：中立裁判者，庭审程序主导者，法律适用权威
- **行为特征**：积极探究、穿透审判、动态控场、权威裁决
- **核心能力**：
  - 法律专业知识与司法经验的综合运用
  - 证据认定能力与争议焦点识别能力
  - 庭审组织与节奏控制能力
  - 复杂法律问题的分析与解决能力

### 律师角色

- **职能定位**：委托人利益代理者，法律服务提供者
- **行为特征**：忠实代理、专业抗辩、策略灵活、目标导向
- **核心能力**：
  - 以委托人最佳利益为导向的法律分析能力
  - 证据组织与质证能力
  - 法庭辩论与说服能力
  - 在法律框架内的策略制定与执行能力

## 使用指南

### 案件准备

1. 在`案件/`目录下创建案件文件夹
2. 按角色创建子文件夹（法官、原告、被告等）
3. 在各角色文件夹中放入相应材料
4. 确保每个律师角色都有`诉讼目的.md`文件

### 庭审操作

- `开始审理：<案件名称>` - 启动庭审
- `继续` - 推进庭审流程
- `结束` - 生成庭审笔录和判决书

### 输出文件

- 庭审笔录：`文书/庭审笔录_YYYYMMDD_HHMMSS.md`
- 判决书：`文书/判决书_YYYYMMDD_HHMMSS.md`
- 争议焦点：`争议焦点.md`
